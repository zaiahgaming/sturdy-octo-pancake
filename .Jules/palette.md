## 2024-07-10 - JSDOM global scope and canvas testing
**Learning:** When using JSDOM to test a vanilla JavaScript game where scripts are executed via `fs.readFileSync` and `runScripts: "dangerously"`, globally defined variables and functions do not automatically attach to the `window` object and must be accessed or manipulated using `window.eval()`. Additionally, JSDOM requires `url: "http://localhost/"` to test `localStorage`.
**Action:** Used `window.eval()` to access variables and invoke game functions during unit tests instead of accessing them directly via the `window` object. Added `url` property to JSDOM init to avoid localStorage SecurityError.
